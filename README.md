# Kiro: From Zero to AI-Native Development

This repository documents my **10-day hands-on journey with Kiro**, where I explore how modern AI-native development works through **spec-driven development**.

Instead of jumping straight into code, this series focuses on building software by first defining *what we want to build*, and then letting Kiro help translate that intent into working applications.

---

##  About This Project

Traditional development starts with setup, boilerplate, and manual implementation.

With Kiro, the workflow shifts toward a more structured approach:

> You define the **spec (what to build)** → Kiro helps generate the **implementation (how it’s built)**

This repo explores that shift through real projects, experiments, and learning logs.

---

## 📅 What This Series Covers

Over 10 days, I’ll be documenting my learning and building journey:

- Day 1: What is Kiro? Understanding spec-driven development  
- Day 2: Installation and first project setup  
- Day 3: Deep dive into specs  
- Day 4: Building a simple app using Kiro  
- Day 5: Comparing Kiro with other AI IDEs  
- Day 6: Why vibe coding fails in production  
- Day 7: Intermediate project building  
- Day 8: Understanding agentic AI in Kiro  
- Day 9: Using Kiro for hackathons  
- Day 10: Final reflection and learning journey  

Each day has its own folder with code, notes, and screenshots.
